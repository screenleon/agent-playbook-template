# Application Implementation

Use this skill for ordinary application changes.

## Check in this order

1. target behavior
2. bounded edit scope
3. state coverage: loading, empty, error, success
4. whether planning or integration help is needed
5. verification path

## Use this skill when

- the task is ordinary product implementation
- the task is frontend or app behavior work without a dedicated visual source
- the task needs code changes but not a new architecture pass
